Data Transfer Object

Verteilung

Data Transfer Object (DTO) fasst in einer verteilten Umgebung (z. B. JEE) zu übertragende Daten in einem neuen Objekt zusammen, um die Anzahl der entfernten Methodenaufrufe zu reduzieren und somit das Netzwerk zu entlasten.

Beispiel

Im Beispiel erlaubt ein CustomerManager den Clients Zugriff auf CustomerEntity-Remote-Objekte. img::data_transfer_object_px.png Bei direkter Verwendung der Entities durch den Client entstehen potenziell viele Netzwerkaufrufe. Daher wird ein CustomerDto eingeführt. img::data_transfer_object_cx.png Die Daten zu einem Kunden werden nun in einem gemeinsamen Call übertragen. img::data_transfer_object_dx.png Ein Client liest/schreibt die einzelnen Datenfelder lokal.

Um das Zusammenspiel im Detail beobachten zu können, setzen Sie den log-level auf DEBUG in der logback.xml und führen den zugehörigen TestCase aus. ref::github(),J2EE
